1.3 Data cleaning & transformation
cleaning quality ▶
Handle missing values in fund and index data
Detect and resolve nulls in NAV price, NAV date, and other critical fields using defined imputation strategies.
For missing NAV price: forward-fill from last available NAV date for same Fund Id
For missing Exchange Code or Currency Code: lookup from fund master reference table
For missing Inception Date: flag as DATA_ISSUE and exclude from since-inception calculations
Log all imputation actions to a data quality audit table

2.2 KPI derivation & aggregate table build
KPI aggregate ▶
Derive period return KPIs
Calculate fund and index returns for standard benchmark periods. These are the primary KPIs shown in the dashboard.
1M return: (NAV_today − NAV_30d_ago) / NAV_30d_ago × 100
1Y, 3Y, 5Y returns using same formula anchored to respective prior date
Since inception return using fund's Inception Date as base NAV date
Calculate same metrics for selected index over identical date windows
Derive deviation: Fund Return − Index Return for each period

Yearly aggregate: annual return for fund and index, year-end NAV
Rolling window aggregate: 30d, 90d, 365d trailing returns updated daily

3.2 Custom measures & calculated KPIs
DAX KPI ▶
Build period return measures in DAX / Tableau
Create dynamic calculated measures for fund and index returns that respond to the date range slicer selection.
DAX: [Fund Return %] = DIVIDE(LASTDATE NAV − FIRSTDATE NAV, FIRSTDATE NAV) × 100
DAX: [Index Return %] = same formula applied to selected index price column
DAX: [Deviation %] = [Fund Return %] − [Index Return %]
Test measures with multiple date range selections and index choices for correctness
